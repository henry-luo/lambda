API Documentation

Complete reference for our REST API. Get started by setting up authentication, then explore available endpoints.

Introduction

Our API is organized around REST principles. All API requests should be made over HTTPS. Authentication is required for all requests.

The API base URL is: https://api.example.com/v1

Authentication

All API requests require an API key in the Authorization header:

Authorization: Bearer YOUR_API_KEY
Note: Store your API keys securely. Never commit them to version control.

Users Endpoint

List Users

GET /users

Retrieve a list of all users.

ParameterTypeDescription
pageintegerPage number (default: 1)
limitintegerItems per page (default: 20)
sortstringSort field (default: -created_at)

Get User

GET /users/{id}

Retrieve a specific user by ID.

Create User

POST /users

Create a new user account.

Error Handling

The API returns standard HTTP status codes. Error responses include a JSON body with details:

{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid request parameters",
    "details": { "email": "Invalid email format" }
  }
}
